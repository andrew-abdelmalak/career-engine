# Cover Letter Rules

> Cross-reference the CV already produced for this application before drafting.

---

## Structure

One page, 250–400 words of body text, three paragraphs:

1. **Opening Hook** (1–2 sentences) — state the role, source, referral, or connection.
2. **Value Proposition** (1–2 paragraphs) — 2–3 of the most relevant achievements, each run through STAR (Situation, Task, Action, Result) as its own beat, not blended into one narrative paragraph. Expand beyond the CV with context and story; a short 2–3 item bullet list is fine if it adds value.
3. **Closing Call to Action** (1 paragraph) — reaffirm fit, thank the reader, note availability, light call to action.

Never duplicate CV bullets verbatim — the cover letter provides narrative context, not a repeat.

---

## STAR Method Per Achievement

Each achievement compressed into one paragraph, four to five sentences total — Action should dominate:

- **Situation** — context, one sentence.
- **Task** — specific responsibility, one sentence.
- **Action** — three to four sentences — approach and method, the most detail of the four.
- **Result** — one sentence, quantified per `CV - Rules.md` Rule 4's honesty standard (real metric where one exists, scope/method where it doesn't).

Prioritize achievements that mirror the job description's stated needs. Use different achievements than what would likely come up in an interview for the same role — keeps material in reserve for follow-up questions.

---

## Format & Professional Standards

- Header matches the CV exactly.
- Include the date.
- Use the hiring manager's name if known, otherwise "Dear Hiring Manager" or "Dear [Department] Team" — never "To whom it may concern."
- State the role early. If the posting gives a reference or requisition code, include it too, kept prominent — not buried mid-paragraph.
- Tone: professional, confident, warm; first person is fine, shouldn't dominate.
- Sign off "Sincerely," "Kind regards," or "Best regards," followed by full name.
- No photos.
- File name: `FirstName_LastName_CoverLetter_CompanyName.pdf`.

---

## Tailoring & Personalization

- Mirror job-description keywords naturally.
- Reference specific company projects, products, or recent developments — search for these if not already known.
- Demonstrate awareness of the company's market position or challenges.
- Connect experience to the company's specific needs, not generic claims.

---

## Critical Mistakes to Avoid

- Generic openings ("I am writing to apply...").
- Wrong company or manager names.
- Overly casual tone or negativity about past employers.
- Discussing salary or benefits unless explicitly requested.

---

## Generation Protocol

External, one-shot document — same stakes profile as the CV. Stop-and-wait, confirm at each stage:

1. **Strategy** — which knowledge-base material and CV facts apply, which rules are active. Confirm.
2. **Audit** — issues vs. the guidelines above, proposed fixes, no draft yet. Confirm.
3. **Draft** — full assembled text. Confirm.
4. **Commit** — final LaTeX output using `Cover Letter - Template.tex` exactly.

Revisions after Commit use a lightweight changelog rather than restaging from Strategy, unless the revision changes which knowledge-base material applies.

Subject to the fact-discipline rule in the project instructions: nothing stated beyond what the knowledge base confirms.
